## How to add a changeset

VIMO uses [Changesets](https://github.com/changesets/changesets) for automated
versioning and changelog generation. **Never bump a package version by hand.**

When your PR changes anything user-facing or contributor-facing, add a changeset:

```bash
npm run changeset
```

Pick the packages you touched and a semver bump (`patch` · `minor` · `major`),
then write a short, human-readable summary. This summary becomes the changelog
entry. On merge to `main`, the release workflow opens a "Version Packages" PR that
bumps versions, updates `CHANGELOG.md`, and (on merge) tags a release.

### Which bump?

- `patch` — bug fixes, docs, CI, chores (no behavior change).
- `minor` — new features, new connectors, new packs (backwards-compatible).
- `major` — breaking changes to the API, schema, or extension contracts.
